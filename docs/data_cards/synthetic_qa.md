# Data Card — Synthetic Advising Q&A

- **Source:** Generated by `drona.finetune.qa_generator` using **local Phi-3.5-mini**
  or the **offline Gemini** path (never the live advising request path)
- **License:** MIT (project-owned)
- **Tier:** `synthetic`
- **Collection method:** programmatic generation, anchored to real entries
- **Volume:** ~500 pairs; ~50 human-reviewed into a gold set

## Contents
Student-style advising questions paired with bias-aware, citation-grounded JSON
answers in the `AdvisingResponse` format.

## Why it's used
LoRA fine-tuning data (Phase 3) to improve adherence to the bias-aware JSON format
and Nepal-first framing, plus an evaluation set. **Never** presented to students as
factual evidence.

## Labelling & anchoring (non-negotiable)
- Every record: `is_synthetic=True` and `synthetic_anchor_ids` linking to the real
  O\*NET/ESCO/posting entries that grounded it.
- Tagged `DataTier.SYNTHETIC`; **never silently mixed** into real retrieval tiers.
- Used for fine-tune/eval only; the base-vs-LoRA ablation checks it actually helps.

## Schema
`qa_id, question, answer_json, anchor_ids[], is_synthetic=True, reviewed: bool,
tier=synthetic`

## Limitations
- Model-generated → may inherit base-model biases; gold review mitigates, not
  eliminates. Diversity bounded by the anchor set and the generator model.
