# Model Card — `phi35-lora-advising`

**Base model:** microsoft/Phi-3.5-mini-instruct (Q4_K_M for serving via Ollama)
**Method:** QLoRA (4-bit NF4) + PEFT LoRA r=16
**Task:** bias-aware academic/career advising (JSON output)
**Version:** 0.1.0  **Created:** 2026-06-09
**License:** Base model license applies (MIT for Phi-3.5); adapter MIT

## Training data
~500 synthetic advising Q&A pairs generated by drona.finetune.qa_generator, grounded in real O*NET/ESCO pathways, Softwarica modules, and (when provided) Nepali job postings. ~50 pairs human-reviewed into a gold set. All synthetic data is labelled and never silently mixed with real data.
- synthetic: **True**
- train examples: 450
- human-reviewed gold: 50

## Hyperparameters
- **r**: 16
- **lora_alpha**: 32
- **epochs**: 3
- **effective_batch_size**: 16
- **learning_rate**: 0.0002
- **max_seq_length**: 2048
- **quantization**: nf4 4-bit

## Intended use
Improve adherence to the bias-aware JSON advising format and Nepal-first framing when used WITH the RAG pipeline (LoRA+RAG). Not a standalone oracle.

## Out of scope
- Use without RAG retrieval context (will hallucinate specifics)
- Any non-advising task
- Live use of cloud LLMs in the request path (local-only invariant)

## Evaluation
- **harness**: drona.finetune.ablation (base+RAG vs LoRA+RAG)
- **metrics**: pathway_count, grounded_rate, nepal_citation_ratio, hedge_frequency, refusal_rate
- **status**: to be filled after Colab T4 training run

## Known limitations
- Trained on synthetic data; gold review mitigates but does not eliminate bias
- Phi-3.5-mini capacity limits complex multi-constraint reasoning
- Adapter must be merged + re-quantised to GGUF for Ollama serving
